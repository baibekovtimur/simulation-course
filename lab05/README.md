# Генератор случайных предсказаний с настраиваемыми вероятностями

## Общее описание

Это Windows Forms приложение на C#, демонстрирующее работу собственного линейного конгруэнтного генератора (LCG) для получения случайных чисел. На его основе реализованы две игры-предсказания:

- **«Скажи “да” или “нет”»** – пользователь задаёт вероятность ответа «Да» (от 0 до 1), программа выдаёт «Да» или «Нет» с учётом указанной вероятности.
- **«Шар предсказаний» (Magic 8‑Ball)** – для каждого из 20 классических ответов можно задать свою вероятность (произвольное неотрицательное число). Перед каждым запросом вероятности нормализуются (сумма приводится к 1), и ответ выбирается в соответствии с заданным распределением.

Все случайные числа генерируются исключительно с помощью собственного LCG-датчика, что позволяет исследовать его статистические свойства.

## Структура проекта

### Содержимое файлов

#### `LcgRandom.cs`

```csharp
namespace RandomGames
{
    public class LcgRandom
    {
        private long _state;
        private readonly long _a = 1103515245;
        private readonly long _c = 12345;
        private readonly long _m = 1L << 31;

        public LcgRandom(long seed) => _state = seed % _m;

        public double NextDouble()
        {
            _state = (_a * _state + _c) % _m;
            return (double)_state / _m;
        }

        public int NextInt(int max) => (int)(NextDouble() * max);
    }
}        
```

#### `MainForm.cs` – содержит основную логику:

## Линейный конгруэнтный генератор (LCG)

LCG – один из простейших генераторов псевдослучайных чисел. Состояние обновляется по формуле:

```
state = (a * state + c) mod m
```

В данной реализации используются параметры, аналогичные стандартному `rand()` в C (glibc):

- `a = 1103515245`

- `c = 12345`

- `m = 2^31` (2 147 483 648)

Каждое значение `state` – целое число в диапазоне `[0, m)`. Метод `NextDouble()` возвращает `state / m` – вещественное число в интервале `[0, 1)`.

Преимущество
 использования собственного LCG – полный контроль над 
последовательностью, возможность смены seed и сравнения с встроенными 
генераторами.

## Алгоритмы выбора с заданными вероятностями

### 1. Игра «Скажи “да” или “нет”»

Вероятность ответа «Да» равна `p` (задаётся пользователем). Ответ «Нет» имеет вероятность `1-p`. Выбор:

- Генерируется случайное число `r ∈ [0, 1)`.

- Если `r < p` → «Да», иначе «Нет».

### 2. Игра «Шар предсказаний»

Каждому
 из 20 ответов пользователь присваивает произвольное неотрицательное 
число (вероятность). Перед каждым запросом программа:

1. Собирает все введённые значения.

2. Вычисляет их сумму `S`.

3. Делит каждое значение на `S` – получает нормализованные вероятности `q_i`.

4. Строит кумулятивные суммы `c[i] = q[0] + ... + q[i]` (причём `c[N-1] = 1`).

5. Генерирует `r ∈ [0, 1)`.

6. Находит наименьший индекс `j`, такой что `r < c[j]` – этот ответ и выбирается.

**Почему это работает:**  
Интервал `[0, 1)` разбивается на N отрезков, длина каждого равна нормализованной 
вероятности соответствующего ответа. Равномерно распределённое число 
попадает в отрезок с вероятностью, равной его длине.

## Интерфейс и работа с программой

### Вкладка «Скажи “да” или “нет”»

- Поле ввода вероятности «Да» (NumericUpDown, от 0 до 1).

- Кнопка «Спросить» – выдаёт случайный ответ.

- Крупная надпись с результатом.

### Вкладка «Шар предсказаний»

- Таблица 4×5 с 20 парами: текст ответа (неизменяемый) и NumericUpDown для задания вероятности.

- Кнопка «Спросить» – нормализует вероятности и выводит выбранный ответ.

- Кнопка «Нормализовать» – вручную приводит вероятности к сумме 1 (можно использовать для проверки).

- Поле с текущим ответом (большой шрифт).

### Общие элементы

- Поле «Seed» – отображает текущее начальное значение LCG.

- Кнопка «Сменить seed» – позволяет задать новое значение seed, после чего генератор пересоздаётся, а ответы сбрасываются.

## Сборка и запуск

### Требования

- .NET SDK 6.0 или выше (или .NET Framework 4.7.2 с поддержкой Windows Forms).

- Visual Studio 2019/2022, VS Code с расширением C#, или любой другой компилятор C#.

### Команды (через dotnet CLI)

1. Создайте новый проект Windows Forms:
   
   ```bash
   dotnet new winforms -n RandomGames
   cd RandomGames
   ```

2. Замените
   содержимое сгенерированных файлов на приведённые выше (Program.cs, 
   MainForm.cs, MainForm.Designer.cs) и добавьте LcgRandom.cs.

3. Соберите и запустите:
   
   ```bash
   dotnet build
   dotnet run
   ```

### Запуск в Visual Studio

- Откройте файл проекта `.csproj` (или создайте новый проект Windows Forms).

- Добавьте файлы в проект.

- Нажмите F5.

## Пример работы

1. Запустите приложение.

2. На вкладке «Скажи “да” или “нет”» установите вероятность «Да» = 0.7.

3. Нажмите «Спросить» несколько раз – примерно 70% ответов будет «Да».

4. Перейдите на вкладку «Шар предсказаний».

5. Измените вероятности некоторых ответов (например, для «Бесспорно» поставьте 5, для «Нет» – 1, остальные – 0.5).

6. Нажмите «Нормализовать» – увидите, как числа привели к сумме 1.

7. Нажимайте «Спросить» – ответы будут появляться с частотой, пропорциональной заданным значениям.

8. Измените seed на другое число – последовательность сбросится.

## Технические детали

- Все случайные числа генерируются LCG – встроенный `Random` не используется.

- Нормализация происходит перед каждым запросом, поэтому пользователь может вводить любые неотрицательные числа.

- Интерфейс построен на `TableLayoutPanel`, что позволяет аккуратно разместить 20 пар элементов без использования `DataGridView`.

- Обработчики событий используют nullable-контекст (`object? sender`), что исключает предупреждения компилятора.

## Заключение

Проект демонстрирует практическое применение линейного конгруэнтного генератора для моделирования случайных событий с произвольным распределением вероятностей. Простота 
LCG позволяет легко понять механизм генерации, а гибкость интерфейса даёт возможность экспериментировать с вероятностями в реальном времени.
