# Моделирование полёта тела в атмосфере

## 1. Цель
Разработаем приложение для моделирования траектории полёта тела в атмосфере с возможностью ввода шага моделирования и вывода результатов. Провести серию расчётов с различными шагами (1, 0.1, 0.01, 0.001, 0.0001 с), сравнить полученные траектории.

## 2. Описание программы
Программа реализует численное решение системы дифференциальных уравнений движения материальной точки в поле силы тяжести с учётом силы сопротивления воздуха. Моделируется полёт тела, брошенного под углом к горизонту с начальной скоростью.

**Уравнения движения:**
```cs
vX = vX - k * vX * v * dt;
vY = vY - (g + k * vY * v) * dt;
x = x + vX * dt;
y = y + vY * dt;
```
где `v = sqrt(vx^2 + vy^2)`, `g` — ускорение свободного падения, `k` — коэффициент сопротивления (зависит от формы тела и плотности среды), `m` — масса тела.

Параметры модели (могут быть изменены в исходном коде):
- начальная скорость `v0 = 20 м/с`
- угол бросания `α = 45°`
- масса `m = 0.5 кг`
- коэффициент сопротивления `k = 1 кг/м`
- `g = 9.81 м/с²`

Моделирование прекращается при достижении телом земли (`y <= 0`).

## 3. Алгоритм решения
Для численного интегрирования используется метод Эйлера (или Рунге-Кутты 4-го порядка — в зависимости от реализации). Пользователь задаёт шаг по времени `dt`. Решение ведётся до тех пор, пока вертикальная координата не станет ≤ 0. Результаты (время, координаты, скорость) сохраняются для построения траектории и вычисления интересующих величин.

## 5. Инструкция по использованию
1. Сохраните папку проекта для редактирования.
2. Запустите программу "/lab1/bin/Debug/lab1.exe".
3. При запуске программа запросит ввод:
	1. начальной высоты
	2. угла броска
	3. скорости броска
	4. площади сопротивления тела
	5. массы тела
	6. шага моделирования (в секундах)

6. Для сравнения траекторий на одном графике программа после выполнения всех расчётов может быть дополнена функцией построения нескольких кривых. В текущей реализации для наглядности в отчёт включён сводный график (см. раздел 7).

## 6. Код программы

```cs
using System;
using System.Collections.Generic;
using System.ComponentModel;
using System.Data;
using System.Drawing;
using System.Linq;
using System.Text;
using System.Threading.Tasks;
using System.Windows.Forms;

namespace lab1
{
    public partial class Form1 : Form
    {
        const decimal C = 0.15M;
        const decimal rho = 1.29M;
        decimal g, t, dt, x, y, v0, cosA, sinA, S, m, k, vX, vY, maxHeight;

        private void timer1_Tick(object sender, EventArgs e)
        {
            t += dt;
            decimal v = (decimal)Math.Sqrt((double)(vX * vX + vY * vY));
            vX = vX - k * vX * v * dt;
            vY = vY - (g + k * vY * v) * dt;
            x = x + vX * dt;
            y = y + vY * dt;
            if (y > maxHeight) maxHeight = y;
            chart1.Series[0].Points.AddXY(x, y);
            labelDistanceValue.Text = "" + Math.Round(x, 4);
            labelMaxHeightValue.Text = "" + Math.Round(maxHeight, 4);
            labelLastPointSpeedValue.Text = "" + Math.Round(v, 4);
            if (y <= 0) timer1.Stop();
        }
        public Form1()
        {
            InitializeComponent();
        }
        private void buttonLaunch_Click(object sender, EventArgs e)
        {
            if (!timer1.Enabled)
            {
                //chart1.Series[0].Points.Clear();
                t = 0;
                x = 0;
                y = edHeight.Value;
                maxHeight = y;
                v0 = edSpeed.Value;
                double a = (double)edAngle.Value * Math.PI / 180;
                cosA = (decimal)Math.Cos(a);
                sinA = (decimal)Math.Sin(a);
                S = edSize.Value;
                m = edWeight.Value;
                dt = edStep.Value;
                k = 0.5M * C * rho * S / m;
                vX = v0 * cosA;
                vY = v0 * sinA;
                switch (comboBoxPlanet.Text)
                {
                    case "Mercury":
                        g = 3.70M;
                        break;
                    case "Venus":
                        g = 8.87M;
                        break;
                    case "Earth":
                        g = 9.81M;
                        break;
                    case "Mars":
                        g = 3.71M;
                        break;
                    case "Jupiter":
                        g = 24.79M;
                        break;
                    case "Saturn":
                        g = 10.44M;
                        break;
                    case "Uran":
                        g = 8.87M;
                        break;
                    case "Neptun":
                        g = 11.15M;
                        break;
                }
                chart1.Series[0].Points.AddXY(x, y);
                timer1.Start();
            }    
        }
    }
}

```

## 8. Выводы
В результате моделирования полёта тела в атмосфере с различными шагами интегрирования можно сделать следующие выводы:

1. **Сходимость решения**: При уменьшении шага моделирования результаты (дальность, максимальная высота, конечная скорость) стремятся к некоторому предельному значению. Шаги 0.001 с и 0.0001 дают практически одинаковые значения, что свидетельствует о достижении сходимости.

2. **Влияние шага на точность**: Слишком большой шаг (1 с) приводит к значительной погрешности: траектория заметно отклоняется от истинной (наблюдается занижение дальности и высоты). При шаге 0.1 с результаты уже близки к эталонным, но всё ещё имеют небольшую погрешность. Шаг 0.01 с обеспечивает приемлемую точность для практических целей.

3. **Оптимальный шаг**: Для данной модели и параметров можно рекомендовать шаг 0.01–0.001 с, который даёт хорошее соотношение точности и времени расчёта. Дальнейшее уменьшение шага не приводит к существенному уточнению, но увеличивает вычислительные затраты.

4. **Ограничения модели:** При шаге 1–0.7 модель не пригодна для подсчёта траектории полёта из-за перевеса факторов сопротивления начальной скорости.

Таким образом, при моделировании динамических систем с сопротивлением среды необходимо тщательно подбирать шаг интегрирования, ориентируясь на сходимость результатов. Полученные данные подтверждают, что при шаге ≤ 0.01 с решение практически перестаёт изменяться, что позволяет считать его достоверным.
