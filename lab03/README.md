## Лабораторная работа «Моделирование лесных пожаров с использованием клеточных автоматов»

Данная программа реализует двумерный клеточный автомат, моделирующий возникновение и распространение лесных пожаров. Проект выполнен на C# с использованием Windows Forms (.NET 8). В модели заложены **4 основных правила** (горение, распространение огня от соседей, случайное возгорание, рост деревьев) и **3 дополнительных правила** (ветер, влажность, температура), которые пользователь может включать и настраивать. Интерфейс позволяет наблюдать за процессом, изменять параметры в реальном времени и вручную редактировать состояние клеток.

---

### Описание модели

Поле представляет собой прямоугольную сетку размером до 200×200 клеток. Каждая клетка может находиться в одном из трёх состояний:

- **Empty** (пусто) – отображается коричневым;

- **Tree** (дерево) – зелёный;

- **Fire** (огонь) – красный.

Эволюция системы происходит дискретно во времени. На каждом шаге новое состояние клетки определяется на основе её текущего состояния, состояний соседей (окрестность Мура – 8 клеток) и набора вероятностных правил. Для повышения производительности отрисовка выполняется через `Bitmap`: при изменении состояния обновляется только соответствующий фрагмент изображения.

### Интерфейс пользователя

Главное окно разделено на две части:

#### Левая область – отображение сетки

- `PictureBox` размером 800×700 пикселей.

- Каждая клетка имеет размер 10×10 пикселей.

- Цвета:
  
  - 🟫 `Empty` – SaddleBrown
  
  - 🌲 `Tree` – ForestGreen
  
  - 🔥 `Fire` – Red

#### Правая панель управления

Панель с вертикальной прокруткой содержит следующие элементы (сверху вниз):

1. **Размер сетки**
   
   - Ползунки *Width* и *Height* (от 10 до 200).
   
   - Кнопка *Resize* – изменяет размер сетки и заполняет её случайными деревьями с текущей вероятностью.

2. **Основные параметры**
   
   - *Start trees* – вероятность того, что клетка изначально будет деревом (0–100%).
   
   - *Lightning (f)* – вероятность случайного возгорания дерева на шаге (0–1%, масштабируется).
   
   - *Regrowth (p)* – вероятность роста дерева на пустой клетке (0–10%).

3. **Дополнительные правила**  
   Каждое правило включается флажком, при активации появляются связанные ползунки:
   
   - **Ветер**
     
     - *Direction (0–7)* – направление ветра (0 – север, 2 – восток, 4 – юг, 6 – запад).
     
     - *Strength* – сила ветра (0–1), увеличивает вероятность распространения в данном направлении.
   
   - **Влажность**
     
     - *Level* – уровень влажности (0–1), снижает вероятность возгорания.
   
   - **Температура**
     
     - *Factor* – коэффициент температуры (0–1), повышает вероятность возгорания.

4. **Режим рисования мышью**
   
   - *Tree* – рисовать деревья.
   
   - *Fire* – поджигать клетки.
   
   - *Empty* – очищать клетки.  
     Рисование происходит при зажатой левой кнопке мыши на поле.

5. **Кнопки управления симуляцией**
   
   - *Start* – запуск автоматического выполнения шагов.
   
   - *Stop* – остановка.
   
   - *Step* – один шаг вручную.
   
   - *Random* – случайное заполнение сетки.
   
   - *Clear* – очистка всех клеток.

6. **Скорость** – ползунок (10–500 мс), задаёт интервал между шагами при автоматическом режиме.

7. **Счётчик поколений** – показывает номер текущего шага.

Все изменения параметров применяются мгновенно – модель использует актуальные значения на каждом следующем шаге.

### Правила клеточного автомата

#### Основные правила

1. **Горение**  
   Если клетка в состоянии `Fire`, на следующем шаге она становится `Empty`.

2. **Распространение огня от соседей**  
   Если клетка в состоянии `Tree` и хотя бы один из её восьми соседей горит, она может загореться.  
   Базовая вероятность загорания от одного горящего соседа равна **1.0** (детерминированное правило), но на неё влияют дополнительные правила.  
   Итоговая вероятность загорания от всех соседей вычисляется как объединение событий:
   
   P=1−i∈горящие соседи∏​(1−pi​)
   
   где pi​ – вероятность от конкретного соседа после модификаторов.

3. **Случайное возгорание (молния)**  
   Даже если ни один сосед не горит, дерево может загореться с вероятностью f (параметр *Lightning*). Это событие происходит после проверки соседей (если дерево ещё не загорелось).

4. **Рост деревьев**  
   Пустая клетка с вероятностью p (параметр *Regrowth*) становится деревом.

### Дополнительные правила

Все дополнительные правила модифицируют базовую вероятность pi​ от одного горящего соседа. Итоговая вероятность от соседа вычисляется как:

pi​=1.0×(1−humidity)×(1+temperature)×(1+wind_bonus)

где:

- humidity – уровень влажности (0..1), включён флажок *Humidity*;

- temperature – температурный коэффициент (0..1), включён флажок *Temperature*;

- wind_bonus – добавляется, если направление от горящего соседа к текущей клетке совпадает с направлением ветра, и ветер включён. Тогда wind_bonus=windStrength, иначе 0.

Полученное значение ограничивается отрезком [0, 1].

#### Метод Step (основная логика)

Метод возвращает список координат клеток, изменивших состояние, что позволяет обновлять только эти области на экране.

```cs
public List<(int x, int y)> Step()
{
    var newGrid = (CellState[,])_grid.Clone();
    var changed = new List<(int, int)>();

    (int dx, int dy) windVec = (0, 0);
    if (WindEnabled) windVec = WindVectors[WindDirection];

    for (int y = 0; y < Height; y++)
        for (int x = 0; x < Width; x++)
        {
            var oldState = _grid[y, x];
            var newState = oldState;

            if (oldState == CellState.Fire)
                newState = CellState.Empty;
            else if (oldState == CellState.Tree)
            {
                double catchProb = 0.0;
                for (int i = 0; i < Neighbors.Length; i++)
                {
                    int nx = x + Neighbors[i].dx;
                    int ny = y + Neighbors[i].dy;
                    if (nx >= 0 && nx < Width && ny >= 0 && ny < Height && _grid[ny, nx] == CellState.Fire)
                    {
                        double p = 1.0;
                        if (HumidityEnabled) p *= (1.0 - HumidityLevel);
                        if (TemperatureEnabled) p *= (1.0 + TemperatureFactor);
                        if (WindEnabled && -Neighbors[i].dx == windVec.dx && -Neighbors[i].dy == windVec.dy)
                            p *= (1.0 + WindStrength);
                        p = Math.Clamp(p, 0.0, 1.0);
                        catchProb = 1.0 - (1.0 - catchProb) * (1.0 - p);
                        if (catchProb >= 1.0) break;
                    }
                }
                if (catchProb < 1.0 && _rand.NextDouble() < LightningProb)
                    catchProb = 1.0;

                newState = _rand.NextDouble() < catchProb ? CellState.Fire : CellState.Tree;
            }
            else // Empty
            {
                newState = _rand.NextDouble() < RegrowthProb ? CellState.Tree : CellState.Empty;
            }

            if (newState != oldState)
            {
                newGrid[y, x] = newState;
                changed.Add((x, y));
            }
            else
                newGrid[y, x] = oldState;
        }

    _grid = newGrid;
    return changed;
}
```

private void UpdateCells(List<(int x, int y)> cells)
{
    using (var g = Graphics.FromImage(gridBitmap))
    {
        foreach (var (x, y) in cells)
        {
            var color = _model.GetCell(x, y) switch
            {
                CellState.Empty => Color.SaddleBrown,
                CellState.Tree  => Color.ForestGreen,
                CellState.Fire  => Color.Red,
                _ => Color.White
            };
            using (var brush = new SolidBrush(color))
                g.FillRectangle(brush, x * _cellSize, y * _cellSize, _cellSize - 1, _cellSize - 1);
        }
    }
    foreach (var (x, y) in cells)
        pictureBox.Invalidate(new Rectangle(x * _cellSize, y * _cellSize, _cellSize, _cellSize));
}

### Запуск программы

1. Убедитесь, что установлен **.NET SDK** (версия 6.0 или выше). Проверить: `dotnet --version`.

2. Скопируйте все файлы проекта (`ForestFire.csproj`, `Program.cs`, `Form1.cs`, `Form1.Designer.cs`, `ForestFireModel.cs`) в одну папку.

3. Откройте терминал в этой папке и выполните:

```bash
dotnet run
```

4. Должно появиться главное окно программы.

Реализованы четыре основных правила (горение, распространение огня от соседей, случайное возгорание, рост деревьев) и три дополнительных (ветер, влажность, температура), которые пользователь может гибко настраивать через графический интерфейс.
